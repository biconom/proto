# Changelog 2026-05-06

## 1. Изменение прав доступа к аналитике (Access Control)
Изменился бит доступа для просмотра аналитики компании. 
- **Ранее** использовался флаг `ADMIN_FINANCE` (бит 24: `1 << 24`).
- **Сейчас** добавлен специализированный флаг `ANALYTICS_COMPANY_VIEW` (бит 30: `1 << 30`).
Теперь для доступа к эндпоинтам аналитики компании клиент должен обладать 30-м битом разрешений. Это обеспечивает более гранулярный контроль доступа без необходимости выдавать полные финансовые права.

## 2. Добавление клиентского API по транзакциям
В клиентский API добавлен новый сервис транзакций (History API).
- **Эндпоинт**: Получение истории транзакций пользователя с пагинацией на основе курсора.
- Данные агрегируются по `group_id`, объединяя связанные проводки (например, переводы с комиссиями).
- Реализована фильтрация по активам (CurrencyId, InventoryId).
- **Обновление proto-типов**: Типы для детализации причин транзакций (`transaction_entry_reason`) перенесены напрямую в пакет `biconom.types.TransactionEntryReason`. Ранее они были во вложенном сообщении `Kind`. Это упрощает импорты и структуру моделей на клиенте.

## 3. Обновление Лидерборда
- Изменена сортировка игроков при равных баллах и одинаковом времени обновления. Теперь при коллизии лидерборд сортирует пользователей по ID **по убыванию** (DESC) вместо возрастания (ASC). То есть пользователи с бо́льшим `player_id` (зарегистрированные позже) будут находиться выше.

## 4. Бонус партнера компании
- В перечисление причин бонусов (`BonusReason`) добавлен новый тип — `company_partner_reward_from_license_purchase`.
- Этот тип отражает моментальное распределение части стоимости лицензии напрямую от компании к партнеру компании.
- В API транзакций это будет отображаться как `Bonus` с соответствующей причиной.
