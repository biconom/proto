# Модели: Geo (карта геометок)

## 1. Описание

**`Geo`** — пространство имён переиспользуемых моделей подсистемы карты геометок дистрибьюторов. Подсистема хранит геометки (`distributor_id → координаты`), а данные иерархии (команда/линия) и баланс WinTime подтягивает из соответствующих подсистем на стороне сервера.

Координаты во внешнем API — `double` (градусы). Внутри хранилища используется fixed-point `i32` (`lat/lon × 1e6`), но эта деталь реализации наружу не утекает.

Модели используются клиентским сервисом `biconom.client.geo.GeoService` (см. `client/geo/geo.md`).

## 2. Модель: `Geo.ScopeOptions`

Область видимости точек на карте по иерархии смотрящего — единообразно с `analytics.ScopeOptions`.

- `Id id`: выбранная область видимости (enum):
  - `UNSPECIFIED = 0` — не задано → ошибка;
  - `PERSONAL = 1` — только сам смотрящий;
  - `TEAM = 2` — поддерево смотрящего (с ограничением глубины `team_depth_limit`);
  - `COMPANY_WIDE = 3` — вся компания (весь проект), без учёта иерархии.
- `optional uint32 team_depth_limit`: максимальная глубина поддерева от смотрящего (только для `TEAM`). `0` → команда не обходится (только сам, если включён `include_personal`); `1` → первая линия; и т.д. Не указано → вся доступная глубина.
- `bool include_personal`: включать ли самого смотрящего в основной список `pins` (в дополнение к `my_pin`). Для `COMPANY_WIDE` не применяется.

> Сам смотрящий в основной список `pins` попадает только при `include_personal = true`; его личная метка всегда отдаётся отдельно (`my_pin`).

## 3. Модель: `Geo.Coordinates`

Географические координаты (градусы) — переиспользуемый тип.

- `double latitude`: широта (градусы, −90..90).
- `double longitude`: долгота (градусы, −180..180).

> Сервер валидирует границы и приводит к внутреннему fixed-point. Используется как запрос `SetPin`.

## 4. Модель: `Geo.Pin`

Геометка дистрибьютора на карте.

- `uint32 distributor_id`: владелец метки.
- `double latitude`: широта (градусы, −90..90).
- `double longitude`: долгота (градусы, −180..180).
- `int64 win_time_balance`: баланс WinTime владельца на момент ответа (знаковый).
- `google.protobuf.Timestamp updated_at`: время последнего обновления метки.

> Если метки нет (например, в `GetDistributorsInfo`), возвращается «пустая» точка с нулевыми координатами и без `updated_at` — фронт отличает такой случай по отсутствию `updated_at`.

## 5. Модель: `Geo.Cluster`

Кластер геометок — плотная группа дистрибьюторов, схлопнутая в одну «точку» карты (автозум).

- `latitude` / `longitude`: центр кластера = **геометрический центр ячейки сетки** (а не среднее координат меток) — кластер визуально привязан к квадрату сетки.
- `distributors_count`: всего дистрибьюторов в кластере.
- `win_time_balance`: суммарный баланс WinTime кластера (знаковый).
- `repeated Pin top`: топ-K меток по балансу (по убыванию) — превью группы для фронта.

> Радиус охвата **не передаётся**: фронт рисует область кластера сам по своей логике (по балансу, количеству дистрибьюторов или иным критериям).

> Сервер группирует прошедшие фильтр (зона + баланс + иерархия) метки по адаптивной сетке: размер ячейки подбирается под целевое число элементов ответа. Ячейка с числом меток выше порога → `Cluster`; разрежённые ячейки отдают метки отдельными `Pin`. Так ответ остаётся компактным даже на карте с ~1 млн точек.

## 6. Связи с другими моделями

- **`Geo.Coordinates`** — вход `SetPin`; сервер приводит к внутреннему fixed-point.
- **`Geo.Pin`** — единица результата `ListPins` (автономные точки) / `GetDistributorsInfo`; также топ-K внутри `Cluster`.
- **`Geo.Cluster`** — плотная группа меток в ответе `ListPins`; содержит агрегаты + топ-K `Pin`.
- **`Geo.ScopeOptions`** — фильтр иерархии в `ListPins`, единообразный с `AnalyticsService`.
- **`Distributor` / `Account`** — полные профили владельцев меток догружаются отдельно через `GetDistributorsInfo` (BFF-паттерн).
