# Модель: Image

## 1. Описание

**`Image`** — модель метаданных загруженного изображения и его обработанных вариантов. Представляет собой «снимок» файла из сервиса хранения изображений (`image_storage_core`), включая информацию об оригинале и готовые к отображению варианты (ресайзы, кропы). Модель не предполагает списочного использования (`List` отсутствует) — изображение всегда привязано к конкретному объекту.

> **Примечание**: Поле `status` (processing/ready) намеренно исключено из proto-модели. Клиентской стороне отдаются только полностью обработанные изображения — статус обработки является внутренним механизмом `image_storage_core`.

## 2. Назначение и решаемые задачи

- **Единая модель для всех изображений**: Аватары, баннеры и другие типы изображений описываются одной моделью, разделяясь по полю `container`.
- **Отображение в клиентских приложениях**: Модель содержит всю информацию, необходимую для выбора оптимального варианта на клиенте (размеры, MIME-тип, вес).
- **Адаптивный выбор варианта**: Карта `variants` позволяет клиенту выбирать подходящий размер по имени (`micro`, `small`, `large` и т.д.) в зависимости от контекста отображения (список, карточка, полноэкранный просмотр).
- **Smart Crop**: Информация о режиме интеллектуального кропа (`smart_crop`, `smart_crop_face_percent`) позволяет клиенту понимать, какие варианты содержат обрезку по лицу.

## 3. Ключевые поля и концепции

| Поле | Тип | Описание |
|------|-----|----------|
| `file_id` | `string` | UUID файла — основной идентификатор, одинаковый для оригинала и всех вариантов. Также используется в `Image.Id` |
| `container` | `string` | Имя контейнера: определяет тип объекта (`ava`, `banner` и т.д.) |
| `object_id` | `uint64` | ID объекта во внешней системе (например, `user.id` для аватаров) |
| `content_type` | `string` | MIME-тип оригинала (например, `image/webp`) |
| `size_bytes` | `uint64` | Размер оригинального файла |
| `created_at` | `Timestamp` | Время создания |
| `variants` | `map<string, Variant>` | Карта обработанных вариантов: ключ — имя (например, `micro`), значение — метаданные варианта |

### Variant (вложенный тип)

| Поле | Тип | Описание |
|------|-----|----------|
| `width` / `height` | `uint32` | Размеры варианта в пикселях |
| `size_bytes` | `uint64` | Размер варианта в байтах |
| `content_type` | `string` | MIME-тип варианта |
| `smart_crop` | `string` | Режим smart crop (`face`, `face_hq`). Пустая строка — стандартный кроп |
| `smart_crop_face_percent` | `uint32` | Процент площади лица в итоговом изображении (1–100). `0` = нет данных |

## 4. Сценарии использования (кейсы)

- **Отображение аватара**: Клиент запрашивает `Image` пользователя → из `variants` берёт `micro` для списка чатов (24×24), `small` для профиля (64×64), `large` для полноэкранного просмотра.
- **Отображение face-crop вариантов**: Вариант с `smart_crop = "face_hq"` гарантирует, что лицо находится в центре кадра — клиент может приоритетно использовать его для превью.
- **Адаптивная загрузка**: Клиент выбирает вариант по `size_bytes` и `width` × `height`, чтобы не загружать тяжёлые файлы на мобильных устройствах.

## 5. Связи с другими моделями

- **`User`** — аватар пользователя хранится как `Image` с `container = "ava"` и `object_id = user.id`.
- **`Distributor`** — аватар дистрибьютора аналогично: `container = "ava"`, `object_id = distributor.id`.
- **`GetAvatarUploadUrlResponse`** (`client.user`) — описывает процесс *загрузки*, тогда как `Image` описывает *результат* обработки.
