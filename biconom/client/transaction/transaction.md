# Client Transaction Service

> API-сервис для получения истории транзакций пользователя, адаптированный под нужды пользовательского интерфейса (BFF — Backend-for-Frontend).

## Контекст

Финансовое ядро использует сложную модель двойной записи (счета компании, комиссии, резервы). Для упрощения работы фронтенда данный сервис инкапсулирует всю внутреннюю бухгалтерию и отдает только готовые **визуальные бизнес-операции**, в которых отфильтрованы служебные проводки и оставлены только изменения, затрагивающие балансы конкретного пользователя.

## Основные сущности

*   **HistoryGroup** — Логическая бизнес-операция (например, "Покупка Лицензии", "Внутренний Перевод", "Торговля"). Группирует в себе одну или несколько изменений балансов. Содержит агрегированный статус операции и общий таймстемп.
*   **HistoryEntry** — Конкретное изменение баланса (списание или зачисление) в рамках группы.
*   **Обогащение данных (BFF Pattern)** — В корне ответа (`HistoryResponse`) возвращаются массивы `accounts`, `distributors` и `slots`. Это справочники всех сущностей, которые упоминаются в истории (например, получатель перевода). Фронтенд может локально сопоставлять ID из проводок с этими массивами для отображения имен и аватарок без дополнительных запросов (избавление от N+1 проблемы).

## Механизм пагинации

API использует инкрементальную курсорную пагинацию (Cursor-based pagination), которая гарантирует стабильную выдачу даже при появлении новых транзакций в реальном времени.

1. **Первый запрос:** Клиент отправляет `HistoryRequest` без `cursor` и задает необходимый `limit` (по умолчанию 100).
2. **Ответ:** Сервис возвращает массив `items` размером до `limit` элементов.
3. **Следующий запрос:** Клиент берет `group_id` из **последнего** элемента массива `items` и передает его в качестве `cursor` (число `uint64`).
4. **Конец списка:** Если API возвращает пустой массив `items` (или массив размером меньше, чем запрошенный `limit`), это означает, что история полностью загружена.

## Фильтрация и Сортировка

*   **Направление (`sort.direction`):** 
    *   `BACKWARD` (по умолчанию) — от новых транзакций к старым.
    *   `FORWARD` — от старых транзакций к новым.
*   **По активам (`filter_assets`):** Позволяет фильтровать историю по конкретным валютам (USDT, WIN_COINS) или инвентарю (Ваучеры). Возвращается вся бизнес-операция (группа), если хотя бы одно изменение баланса в ней затронуло указанный актив.
*   **По категориям (`filter_kinds`):** Позволяет фильтровать по типам операций (например, только `TRANSFER` или `BONUS`).

## Безопасность и Access Control

*   Доступ к сервису разрешен только для авторизованных пользователей с валидным сессионным токеном.
*   В ответ попадают строго только те изменения балансов (`HistoryEntry`), которые принадлежат запрашивающему аккаунту. Данные других участников платформы надежно изолированы.
