# Сервис: UserService

## 1. Описание

**`UserService`** — это клиентский сервис, который предоставляет методы для работы с данными, специфичными для текущего аутентифицированного пользователя. Он позволяет запрашивать информацию, которая не относится напрямую к аккаунту или балансам, например, права доступа.

## 2. Описание методов (RPC)

### `rpc GetPermissions(google.protobuf.Empty) returns (GetPermissionsResponse)`
- **Назначение**: Получить битовые маски прав доступа для текущего пользователя.
- **Использование**: Вызывается клиентским приложением для того, чтобы понять, какие действия (например, создание заявки на вывод, просмотр истории) доступны пользователю. Это позволяет динамически отображать или скрывать элементы интерфейса.
- **Параметры**: Принимает `google.protobuf.Empty`, так как ID пользователя берется из контекста авторизации.
- **Ответ**: Возвращает `GetPermissionsResponse`, содержащий:
  - `personal_permissions_bit_mask`: Персональные права пользователя (что он может делать с собой и своими данными).
  - `delegated_permissions_bit_mask`: Делегированные права (что он может делать над другими пользователями, если применимо).
- **Авторизация**: Требуется.

### `rpc GetTelegramBotLink(google.protobuf.Empty) returns (GetTelegramBotLinkResponse)`
- **Назначение**: Запросить ссылку на телеграм-бота для подключения.
- **Использование**: Клиентское приложение вызывает этот метод, чтобы получить уникальную ссылку для текущего пользователя. При переходе по этой ссылке пользователь сможет привязать свой аккаунт к Telegram-боту.
- **Параметры**: Принимает `google.protobuf.Empty`, так как ID пользователя определяется из контекста авторизации.
- **Ответ**: Возвращает `GetTelegramBotLinkResponse`, содержащий:
  - `link`: Уникальная ссылка для подключения к боту.
- **Авторизация**: Требуется.
