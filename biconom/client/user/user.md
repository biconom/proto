# Сервис: UserService

## 1. Описание

**`UserService`** — это клиентский сервис, который предоставляет методы для работы с данными, специфичными для текущего аутентифицированного пользователя. Он позволяет запрашивать информацию, которая не относится напрямую к аккаунту или балансам, например, права доступа.

## 2. Описание методов (RPC)

### `rpc GetPermissions(google.protobuf.Empty) returns (GetPermissionsResponse)`
- **Назначение**: Получить битовые маски прав доступа для текущего пользователя.
- **Использование**: Вызывается клиентским приложением для того, чтобы понять, какие действия (например, создание заявки на вывод, просмотр истории) доступны пользователю. Это позволяет динамически отображать или скрывать элементы интерфейса.
- **Параметры**: Принимает `google.protobuf.Empty`, так как ID пользователя берется из контекста авторизации.
- **Ответ**: Возвращает `GetPermissionsResponse`, содержащий:
  - `personal_permissions_bit_mask`: Персональные права пользователя (что он может делать с собой и своими данными).
  - `delegated_permissions_bit_mask`: Делегированные права (что он может делать над другими пользователями, если применимо).
- **Авторизация**: Требуется.

### `rpc GetTelegramBotLink(google.protobuf.Empty) returns (GetTelegramBotLinkResponse)`
- **Назначение**: Запросить ссылку на телеграм-бота для подключения.
- **Использование**: Клиентское приложение вызывает этот метод, чтобы получить уникальную ссылку для текущего пользователя. При переходе по этой ссылке пользователь сможет привязать свой аккаунт к Telegram-боту.
- **Параметры**: Принимает `google.protobuf.Empty`, так как ID пользователя определяется из контекста авторизации.
- **Ответ**: Возвращает `GetTelegramBotLinkResponse`, содержащий:
  - `link`: Уникальная ссылка для подключения к боту.
- **Авторизация**: Требуется.

### `rpc DeleteTelegram(google.protobuf.Empty) returns (google.protobuf.Empty)`
- **Назначение**: Удалить привязку телеграм от аккаунта.
- **Использование**: Клиентское приложение вызывает этот метод, чтобы отвязать аккаунт текущего пользователя от Telegram-бота. После этого пользователь перестанет получать уведомления и не сможет использовать команды бота до повторной привязки.
- **Параметры**: Принимает `google.protobuf.Empty`, так как операция выполняется для текущего аутентифицированного пользователя.
- **Ответ**: Возвращает `google.protobuf.Empty` в случае успешного удаления.
- **Авторизация**: Требуется.

### `rpc GetAvatarUploadUrl(google.protobuf.Empty) returns (GetAvatarUploadUrlResponse)`
- **Назначение**: Получить временную ссылку (pre-signed URL) для загрузки файла аватарки пользователя.
- **Использование**: Вызывается перед тем, как клиентское приложение начнет загрузку изображения. Клиент выполняет загрузку напрямую в хранилище (например, AWS S3) через полученную ссылку (обычно посредством PUT-запроса). Все ключи авторизации и подписи уже вшиты в `upload_url`. Дополнительные заголовки авторизации не требуются.
- **Параметры**: `google.protobuf.Empty`. Идентификатор определяется по сессии.
- **Ответ**: Возвращает `GetAvatarUploadUrlResponse`:
  - `upload_url`: Сгенерированная временная ссылка для загрузки.
  - `expires_at`: Временная метка, когда ссылка перестанет быть действительной (истечет срок её жизни). Позволяет клиенту корректно обработать устаревание ссылки.
  - `allowed_formats`: Список допустимых форматов файлов (например: `jpg`, `jpeg`, `png`, `webp`, `bmp`, `gif`, `tiff`). Клиент должен проверить расширение файла перед загрузкой.
  - `max_file_size`: Максимальный размер файла в байтах (`uint64`). Клиент должен проверить размер файла перед загрузкой (например, 5 МБ = 5 242 880 байт).
- **Авторизация**: Требуется.

### `rpc ResetAvatar(google.protobuf.Empty) returns (google.protobuf.Empty)`
- **Назначение**: Сбросить или удалить текущую аватарку пользователя.
- **Использование**: Клиент вызывает данный метод для возврата к аватарке по умолчанию. Старое изображение становится неактивным или удаляется.
- **Параметры**: `google.protobuf.Empty`.
- **Ответ**: `google.protobuf.Empty` по факту успешного удаления.
- **Авторизация**: Требуется.
