# WinTime Client Service

> API-сервис игрового токен-баланса **WinTime**: текущий баланс пользователя и история начислений/списаний.

## Контекст

WinTime — отдельная подсистема игрового токена. Каждый дистрибьютор имеет кошелёк WinTime
(`balance` + порядковый номер последней транзакции `seq`). Баланс пополняется поминутным
пассивным начислением, реферальными бонусами за личников и бонусами за закрытие уровней
в маркетинговом дереве, а также корректируется администратором.

Сервис предоставляет авторизованному пользователю доступ **только к собственному** балансу:
владелец определяется по сессии (`distributor_id` из `executor`), явный идентификатор в запросах
не передаётся.

## Основные сущности

* **`WinTime.Balance`** — текущий баланс кошелька: `amount` (знаковый `int64`, целые токены) и
  `seq` (порядковый номер последней транзакции).
* **`WinTime.Transaction`** — одна операция изменения баланса: `seq` (курсор), `amount`
  (`+` начисление / `−` списание), `created_at` (время) и типизированные детали `oneof details`
  (по варианту на каждый вид начисления).
* **`BalanceResponse`** — единая модель ответа обоих методов: `balance` + массив `items`
  (транзакции, по умолчанию от новых к старым) + BFF-справочники `slots` / `distributors` /
  `accounts`.

## Детали транзакций (`Transaction.details`, oneof)

| Вариант | Смысл | Поля | Обогащение |
|---|---|---|---|
| `admin_adjust` | Ручная корректировка администратором | — | — |
| `passive_bonus` | Поминутное пассивное начисление | `multiplier` | — |
| `referral_bonus` | Реферальный бонус за личника | `child_distributor_id`, `multiplier` | `distributors` / `accounts` |
| `slot_depth_filled_bonus` | Бонус за закрытие уровня в дереве | `slot_id`, `depth` | `slots` / `distributors` / `accounts` |

## Обогащение данных (BFF-паттерн)

В деталях транзакций лежат только идентификаторы (`slot_id`, `child_distributor_id`). Полные
модели возвращаются дедуплицированными массивами в корне `BalanceResponse`:

* **`slots`** — слоты, упомянутые в `slot_depth_filled_bonus`;
* **`distributors`** — дистрибьюторы личников (`referral_bonus`) и владельцы упомянутых слотов;
* **`accounts`** — аккаунты этих дистрибьюторов.

Фронт локально сопоставляет id из деталей с этими справочниками — без N+1 запросов. Видимость
сущностей рассчитывается относительно запрашивающего (`relationship_state`).

## Методы

### `GetBalance`

Возвращает текущий баланс и последние **10** транзакций (от новых к старым). Удобно для виджета
баланса с превью истории.

### `ListTransactions`

Возвращает баланс и выборку транзакций (до **1000** за запрос). Параметры:

* **`cursor`** (`optional uint32`) — `seq` последней полученной транзакции (exclusive); отсутствие
  курсора означает выборку с начала/конца в зависимости от направления.
* **`sort.direction`** — `BACKWARD` (по умолчанию, новые → старые) или `FORWARD` (старые → новые).
* **`sort.limit`** — размер выборки; `0` или превышение трактуется как `1000` (жёсткий потолок).

Дальнейшая пагинация (подгрузка следующих страниц по `seq` последнего элемента) выполняется
**на стороне клиента**.

## Безопасность и Access Control

* Доступ только для авторизованных пользователей с валидным сессионным токеном
  (`Permission::WALLET_VIEW`). Гостевые и confirmation-токены отклоняются.
* Возвращается строго баланс и история **запрашивающего** дистрибьютора; данные других
  участников изолированы.
