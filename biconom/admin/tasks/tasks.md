# Service: TasksService

## 1. Описание
Сервис `TasksService` предоставляет административные методы для выполнения массовых операций и фоновых задач обслуживания системы.
Включает в себя методы для принудительной активации лицензий и других пакетных процессов.

## 2. Методы API

### `LicenseActivateAllInactive`
- **Назначение**: Запускает процесс активации всех маркетинговых лицензий, находящихся в статусе `inactive`.
- **Входные параметры**: Нет (`LicenseActivateAllInactiveRequest` пустой).
- **Возвращаемое значение**: Структура `LicenseActivateAllInactiveResponse` содержащая:
    - `success_count` (uint32): Количество успешно активированных лицензий.
    - `failure_count` (uint32): Количество лицензий, активация которых завершилась ошибкой.
- **Побочные эффекты**:
    - Изменяет статус лицензий с `inactive` на `active` (или соответствующий активный статус).
    - Может инициировать начисление бонусов или другие связанные процессы в зависимости от бизнес-логики активации лицензии.
    - Создает записи в аудит-логе о массовой активации.

### `MarketingProcessPaymentObligations`
- **Назначение**: Запускает процесс расстановки обязательств на выплату по маркетингу (начисление бонусов) для конкретного слота.
- **Входные параметры**: `biconom.types.Slot.Id` (Идентификатор слота).
- **Возвращаемое значение**: `google.protobuf.Empty` (пустой ответ при успешном принятии задачи).
- **Побочные эффекты**:
    - Инициирует расчет и создание платежных обязательств (бонусов) для указанного слота и его аплайнов.
    - Может привести к изменению балансов пользователей.
    - Создает записи о начислениях в логах.

## 3. Права доступа и безопасность
- **Требуемые права**: Требуются права администратора с доступом к управлению лицензиями (`LICENSE_MANAGE` или `SUPER_ADMIN`).

## 4. Сценарии использования
- **Массовая активация**: Используется администратором для ручного запуска активации лицензий, которые по каким-то причинам не активировались автоматически, или для проведения маркетинговых акций.
- **Устранение сбоев**: Позволяет "протолкнуть" застрявшие в неактивном состоянии лицензии после исправления системных ошибок.
