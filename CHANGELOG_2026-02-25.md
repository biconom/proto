# Changelog: 25 февраля 2026

## biconom/types/image.proto (НОВЫЙ ФАЙЛ)

Новая модель данных **`Image`** — метаданные загруженного изображения и его обработанных вариантов.

### Image
- `file_id` (`=1`, string) — UUID файла (первичный идентификатор).
- `container` (`=2`, string) — имя контейнера (тип объекта: `ava`, `banner`).
- `object_id` (`=3`, uint64) — ID объекта во внешней системе.
- `content_type` (`=4`, string) — MIME-тип оригинала (например, `image/webp`).
- `size_bytes` (`=5`, uint64) — размер оригинального файла в байтах.
- `created_at` (`=6`, Timestamp) — время создания.
- `variants` (`=7`, map<string, Variant>) — карта обработанных вариантов по имени.

### Image.Id
- `file_id` (`=1`, string, oneof) — UUID файла.

### Image.Variant
- `width` (`=1`, uint32) — ширина в пикселях.
- `height` (`=2`, uint32) — высота в пикселях.
- `size_bytes` (`=3`, uint64) — размер варианта в байтах.
- `content_type` (`=4`, string) — MIME-тип варианта.
- `smart_crop` (`=5`, string) — режим smart crop (`face`, `face_hq`). Пустая строка = стандартный кроп.
- `smart_crop_face_percent` (`=6`, uint32) — процент площади лица (1–100). `0` = нет данных.

> **Для верстальщика**: Варианты — это карта (ключ = имя: `micro`, `small`, `large` и т.д.). Клиент выбирает вариант по имени в зависимости от контекста (список = `micro` 24×24, профиль = `small` 64×64, полный = `large`).

## biconom/client/user/user.proto

### UserService — новые методы

| Метод | Описание |
|-------|----------|
| `GetAvatarUploadUrl` | Получить временную pre-signed ссылку для загрузки аватарки. |
| `ResetAvatar` | Сбросить (удалить) текущую аватарку пользователя. |

### GetAvatarUploadUrlResponse (новое)
- `upload_url` (`=1`, string) — pre-signed URL для PUT-запроса загрузки файла.
- `expires_at` (`=2`, Timestamp) — когда ссылка истекает.
- `allowed_formats` (`=3`, repeated string) — допустимые форматы файлов (`jpg`, `jpeg`, `png`, `webp`, `bmp`, `gif`, `tiff`).
- `max_file_size` (`=4`, uint64) — максимальный размер файла в байтах (например, 5 МБ = 5 242 880).

> **Для верстальщика**: Перед загрузкой валидировать файл на клиенте:
> 1. Проверить расширение по `allowed_formats`.
> 2. Проверить размер по `max_file_size`.
> 3. Загрузить файл PUT-запросом на `upload_url` (без дополнительных заголовков авторизации).
> 4. Обработать истечение ссылки по `expires_at`.

## Инструкция: работа с изображениями (Image Storage)

### Загрузка изображения

1. Вызвать `UserService.GetAvatarUploadUrl()` — получить `upload_url` и `token` (токен содержится в URL).
2. Отправить **POST** `https://storage.win2partners.com/upload/{token}` с файлом в поле **`file`** (multipart/form-data).
3. Перед загрузкой валидировать на клиенте:
   - Формат файла по `allowed_formats`.
   - Размер файла по `max_file_size`.
   - Срок действия ссылки по `expires_at`.

> ⚠️ **Ссылка на загрузку одноразовая.** После успешной загрузки файла токен считается использованным. Повторный запрос по той же ссылке вернёт ошибку. Для новой загрузки необходимо заново вызвать `GetAvatarUploadUrl()`.

### Получение изображения

URL для получения варианта:
```
https://storage.win2partners.com/file/{file_id}/{variant_name}
```

Пример: если `file_id = "9be80338-f316-4404-becb-c9c5d88fade2"` и нужен вариант `small`:
```
https://storage.win2partners.com/file/9be80338-f316-4404-becb-c9c5d88fade2/small
```

### Smart Crop (интеллектуальная обрезка)

| Значение `smart_crop` | Описание |
|------------------------|----------|
| `"face_hq"` | Найдено лицо на фото. Область захвачена за пределы лица — **качественный кроп без растяжения**. Рекомендуется для аватарок. |
| `"face"` | Найдено лицо на фото. Есть **риск растяжения**, если лицо было маленьким на исходном изображении. |
| `""` (пустая строка) | Стандартный кроп без определения лица. |

**`smart_crop_face_percent`** — сколько процентов площади итогового изображения занимает лицо.
Например, `smart_crop_face_percent = 30` означает, что лицо занимает 30% площади фото. Чем выше значение — тем крупнее лицо в кадре.
